What is Crucible?

MCP servers give AI assistants the ability to use external tools — but managing them is painful. You have to build, configure, and run each server individually, across different clients and machines.


Crucible solves this. Paste a GitHub URL and it automatically builds, deploys, and exposes the server as an SSE endpoint. Manage everything from a single UI. No external SaaS required — it runs entirely on your own infrastructure.

Key Features

🔗

Build from GitHub URL

Paste a repository URL and Crucible builds and deploys it automatically. No pre-built images required.

🔒

Private Repository Support

Works with private GitHub repositories. Your proprietary MCP servers stay private.

🔄

Automatic stdio to SSE

stdio-only servers are automatically exposed as SSE endpoints via mcp-proxy for remote access.

🖥

Management UI

Monitor status, restart, and remove servers from your browser.

🛡

Secure by Default

Docker Socket Proxy limits Docker operations to minimum privileges.

🏠

Fully Self-hosted

No external dependencies. Everything runs on your machine, your network, under your control.

How it works

1

Paste a GitHub URL

Enter the URL of any MCP server repository — public or private.

2

Auto build & deploy

Crucible builds a Docker container and starts the server with an SSE endpoint.

3

Connect from any client

Add the SSE URL to Claude Code, Cursor, or any MCP-compatible client.

Quick Start

# Clone the repository
git clone https://github.com/kumagallium/Crucible.git
cd Crucible

# Configure environment variables
cp .env.example .env
chmod 600 .env

# Start Crucible
docker compose up -d

# Access the UI at http://127.0.0.1:8081
