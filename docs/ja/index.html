Crucible

あなたの MCP サーバーを、あなたのインフラで、あなたの管理下に。

MCP サーバーのビルド・デプロイ・管理を一箇所で行う、 セルフホスト型レジストリ。

はじめる GitHub

Crucible とは？

MCP サーバーは AI アシスタントに外部ツールを使う力を与えます。しかし、その管理は大変です。 サーバーごとにビルド・設定・起動を行い、クライアントやマシンごとに個別管理が必要です。


Crucible はこの問題を解決します。GitHub URL を貼るだけで、サーバーを自動ビルド・デプロイし、 SSE エンドポイントとして公開。すべてを一つの管理画面で操作できます。 外部 SaaS は不要 — すべてあなたのインフラ上で完結します。

こんな方に

こんな状況に心当たりはありませんか？

個人開発者 — MCP サーバーの管理疲れ
Before
  • Claude Code、Cursor、Windsurf など複数のクライアントで MCP サーバーを使っている
  • サーバーごとに手動でビルド、起動、設定ファイルを書いている
  • 新しい MCP サーバーを GitHub で見つけても「試すの面倒だな」でスルーしてしまう
  • stdio のサーバーをリモートから使いたいが、mcp-proxy の設定が面倒
After
  • GitHub URL を貼るだけ。ビルド・デプロイ・SSE エンドポイント公開まで自動
  • 全サーバーを 1 つの管理画面で確認・再起動・削除
  • stdio サーバーも自動で SSE 化されるのでリモートからすぐ使える
  • 「気になったら URL を貼って試す」ができるようになる
小規模チーム — MCP ツールの社内共有
Before
  • チーム内で「この MCP サーバー便利だよ」と共有しても、各自がローカルでセットアップしないといけない
  • 人によって環境が違い、「動かない」が頻発する
  • 同じサーバーを全員が個別に起動するのは無駄
  • SaaS 型の MCP ホスティングは社内ポリシー的に使えない
After
  • チーム共有のサーバーに Crucible をデプロイ
  • 1 人がサーバーを登録すれば、他のメンバーは SSE URL を追加するだけで使える
  • 環境差異の問題がなくなる（Docker で統一）
  • 外部 SaaS 不要。VPN 内で完結
セキュリティ重視の組織 — AI ツールアクセスの主権
Before
  • MCP サーバーを外部 SaaS に置くと、ツール呼び出しのデータが外部を経由する
  • どのサーバーが何にアクセスしているか把握しづらい
  • 情報セキュリティポリシー上、外部ホスティングの承認プロセスが重い
  • かといって自前で管理するインフラ構築コストも高い
After
  • 全て自組織のインフラ上で動作。データが外部に出ない
  • Docker Socket Proxy で Docker 操作を最小権限に制限
  • 管理 UI でサーバーの状態を一覧把握
  • docker compose up -d で構築完了。専用インフラ不要

主な機能

🔗

GitHub URL からビルド

リポジトリ URL を貼るだけで自動ビルド＆デプロイ。ビルド済みイメージは不要。

🔒

プライベートリポジトリ対応

プライベート GitHub リポジトリにも対応。独自の MCP サーバーを公開する必要なし。

🔄

stdio → SSE 自動変換

stdio サーバーも mcp-proxy で自動的に SSE エンドポイントとして公開。

🖥

管理 UI

サーバーの状態確認・再起動・削除をブラウザから操作。

🛡

セキュア設計

Docker Socket Proxy で Docker 操作を最小権限に制限。

🏠

完全セルフホスト

外部依存なし。すべてあなたのマシン、あなたのネットワーク内で完結。

仕組み

1

GitHub URL を入力

MCP サーバーのリポジトリ URL を入力。パブリック・プライベート両対応。

2

自動ビルド＆デプロイ

Docker コンテナをビルドし、SSE エンドポイント付きでサーバーを起動。

3

MCP クライアントから接続

Claude Code、Cursor など、お好みの MCP クライアントから SSE URL で接続。

クイックスタート

# リポジトリをクローン
git clone https://github.com/kumagallium/Crucible.git
cd Crucible

# 環境変数を設定
cp .env.example .env
chmod 600 .env

# Crucible を起動
docker compose up -d
